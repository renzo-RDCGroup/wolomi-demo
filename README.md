# Wolomi Landing Page Demos — by Renzo

Two trust-led, mobile-first intake concepts that replace the current ~18-field eligibility form with a short qualifier (county, life-stage, Medi-Cal status, consent) and collect the rest after the mom is engaged.

- `index.html` — hub page comparing both concepts (share this link in the interview)
- `concept-a-progress.html` — stepped form with a progress bar
- `concept-b-conversational.html` — chat-style intake with a "Mama Companion"

All three are standalone HTML — no build step, no backend, nothing stored. Submissions are simulated.

---

## Host it free on GitHub Pages (≈3 minutes)

1. Go to **github.com** → sign in (or create a free account) → click **New repository**.
   - Name it e.g. `wolomi-demo`, set it to **Public**, click **Create repository**.
2. On the new repo page, click **uploading an existing file**.
   - Drag in all three `.html` files (and this README). Click **Commit changes**.
3. Go to **Settings → Pages** (left sidebar).
   - Under **Build and deployment → Source**, choose **Deploy from a branch**.
   - Branch: **main**, folder: **/ (root)** → **Save**.
4. Wait ~1 minute, refresh. GitHub shows your live URL:
   - `https://YOUR-USERNAME.github.io/wolomi-demo/`
5. Share these links with the client:
   - Hub: `https://YOUR-USERNAME.github.io/wolomi-demo/`
   - Concept A: `.../concept-a-progress.html`
   - Concept B: `.../concept-b-conversational.html`

### Optional: a cleaner URL
Add a free custom domain in **Settings → Pages → Custom domain** (e.g. a subdomain like `demo.yourdomain.com`) if you want it on your own brand.
